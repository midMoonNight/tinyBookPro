## 1. 产品概述

极简记账本 Pro 是一个面向个人用户的日常记账微信小程序。用户可以在不登录的情况下直接记录收入和支出，数据仅保存在本机；用户使用微信一键登录后，记账数据同步保存到云端数据库。

第一版目标是完成一个简洁、稳定、易用的个人记账闭环：快速记账、查看今日概览、查看日/月统计、设置每月总预算，并在小程序内展示预算使用提醒。

[2026-07-14 新增] 第二版目标是在不增加复杂账户体系的前提下，补齐账单管理、分类管理、统计分析和数据导出能力，让用户能够更方便地查找、修正和使用自己的历史记账数据。

## 2. MVP 目标

### 2.1 核心目标

- 用户打开小程序后，可以快速了解今天的收入、支出和结余。
- 用户可以快速记录一笔收入或支出，并选择固定分类。
- 用户可以查看按日、按月统计的收支情况。
- 用户可以设置每月总预算，并看到预算使用进度和超支提醒。
- 用户未登录时也能正常使用记账功能。
- 用户登录后，可以将本地数据同步到云端，后续以云端数据为准。
- [2026-07-14 新增] 用户可以查看并筛选全部历史账单。
- [2026-07-14 新增] 用户可以编辑已保存的账单。
- [2026-07-14 新增] 用户可以管理自己的收支分类。
- [2026-07-14 新增] 用户可以查看更完整的月度统计和月份对比。
- [2026-07-14 新增] 用户可以导出个人账单数据。

### 2.2 非目标

第一版暂不支持以下能力：

- 分类预算。
- ~~自定义收支分类。~~ [2026-07-14 删除：第二版纳入自定义分类能力。]
- 多账本。
- 多人共享账本。
- 自动读取微信支付账单。
- 微信服务通知或模板消息提醒。
- 多币种。
- 发票、图片、附件管理。
- 复杂资产、负债、账户管理。
- [2026-07-14 新增] 第二版仍不支持分类预算、账户余额管理、周期记账、多人共享和票据附件。

## 3. 用户与使用场景

### 3.1 目标用户

需要记录个人日常收入和支出的普通用户。

### 3.2 核心场景

- 用户消费后，打开小程序快速记录一笔支出。
- 用户收到工资、奖金或兼职收入后，记录一笔收入。
- 用户每天查看当天收支情况。
- 用户每月查看整体支出趋势和分类占比。
- 用户设置每月总预算，并关注本月支出是否接近或超过预算。
- 用户未登录时先试用，后续登录并同步本机数据。

## 5. 功能需求

### 5.1 记账 Tab

#### 5.1.1 今日概览

用户打开小程序后，默认进入记账 Tab，页面展示今天的收支概览。

展示内容：

- 今日收入
- 今日支出
- 今日结余
- 本月支出
- 本月预算进度，用户已设置预算时展示

规则：

- 今日收入统计当天所有收入记录。
- 今日支出统计当天所有支出记录。
- 今日结余 = 今日收入 - 今日支出。
- 本月支出统计当前自然月内所有支出记录。
- 预算进度根据本月支出和本月总预算实时计算。

#### 5.1.2 快速记一笔

用户可以快速新增一条记账记录。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 类型 | 是 | 支出或收入 |
| 金额 | 是 | 必须大于 0 |
| 分类 | 是 | 第一版使用固定分类；第二版可选择系统分类或用户自定义分类 |
| 日期 | 是 | 默认今天，可修改 |
| 备注 | 否 | 用户可填写简单说明 |

交互规则：

- 默认类型为支出。
- 默认日期为当天。
- ~~金额不能为空，且必须为大于 0 的数字。~~ [2026-07-16 调整：金额必须大于 0、不超过 99,999,999.99，且最多保留两位小数。]
- [2026-07-16 新增] 备注最多 40 个字符；中文、英文、数字和表情均按用户可见字符计数。输入或粘贴超限内容时不得静默截断，需要显示实时字数和错误状态，保存时明确提示并阻止提交。
- ~~新增成功后，返回记账 Tab 并刷新今日概览、今日记录和预算进度。~~ [2026-07-14 调整：新增成功后刷新今日概览、最近记录和预算进度。]

#### 5.1.3 分类

~~第一版分类固定，不支持用户自定义。~~ [2026-07-14 调整：第一版保持固定分类，第二版增加用户自定义分类。]

支出分类：

- 餐饮
- 交通
- 购物
- 居住
- 娱乐
- 医疗
- 学习
- 其他

收入分类：

- 工资
- 奖金
- 兼职
- 投资
- 其他

#### 5.1.4 最近记录列表

记账 Tab 展示当天的记账记录列表。

~~[2026-07-14 补充] “今日记录”标题右侧增加蓝色文字入口“全部”，点击进入全部账单二级页面。~~ [2026-07-14 调整：“今日记录”改为“最近记录”，入口位置和文字“全部”保持不变。]

- 最近记录按记录创建时间 `createdAt` 倒序展示最近 5 笔，不限账单的记账日期。
- 登录用户打开记账 Tab 时从云端按 `createdAt` 拉取最近 5 笔，确保换设备后展示口径一致。
- 用户补记历史日期后，新账单立即展示在最近记录首条，避免用户误认为保存失败。
- 当天账单显示录入时间，非当天账单显示实际记账日期。
- 今日概览仍只统计记账日期为当天的账单，不受最近记录展示范围影响。

展示内容：

- 类型
- 分类
- 金额
- 备注，存在时展示
- 记录时间或日期

第一版支持：

- 查看当天记录。
- 删除记录。

第一版暂不要求支持：

- ~~编辑记录。~~ [2026-07-14 删除：第二版支持编辑账单。]
- ~~搜索记录。~~ [2026-07-14 删除：第二版通过全部账单筛选和备注关键词查询实现。]
- 批量操作。

#### 5.1.5 全部账单

[2026-07-14 新增]

用户可以从记账 Tab 进入“全部账单”，查看当前账号或本机保存的历史记录。

[2026-07-14 补充后调整] “全部账单”不是新的数据类型，也不单独创建集合。它展示的是用户通过“快速记一笔”录入并保存在 `records` 集合或本地缓存中的全部有效收入、支出记录。最近记录、按日统计、按月统计和全部账单都是同一批账单数据的不同查询和展示方式。

单条账单记录的数据包括：

- 收支类型。
- 金额。
- 分类 ID 和分类名称快照。
- 记账日期。
- 备注。
- 创建时间和更新时间。
- 同步和软删除所需的内部字段；内部字段不直接展示给用户。

展示规则：

- 默认按记账日期倒序排列，同一天内按创建时间倒序排列。
- 列表按日期分组，日期标题展示当日收入、支出或净收支摘要。
- 单条记录展示类型、分类、金额、备注和记账时间。
- 已软删除的记录不展示。
- 首次加载最近 30 天或最近 100 条，继续滚动时分页加载，避免一次读取全部历史数据。

筛选条件：

- 时间范围：本月、上月、近 3 个月、自定义日期范围；单次最多覆盖连续 3 个自然月。
- 类型：全部、收入、支出。
- 分类：根据当前启用分类和历史账单分类生成选项。
- 备注关键词：按用户输入的关键词匹配备注。

筛选规则：

- 多个筛选条件同时存在时按交集查询。
- ~~开始和结束日期跨越的自然月份不得超过 3 个；日期选择器动态限制可选范围，查询前再次校验。~~ [2026-07-16 调整：日期选择器不互相限制可选日期，用户可以先后自由选择开始和结束日期；若开始日期晚于结束日期或跨越超过连续 3 个自然月，保留已选日期并在筛选区持续显示校验提示，不执行查询，调整为有效范围后自动查询。]
- 分类筛选使用两列级联选择：第一列为全部、支出、收入，第二列只展示所选类型下的分类；选择全部时第二列只显示全部分类。
- 顶部类型筛选按钮与分类级联选择保持同步，任一入口改变类型时清空不再适用的分类条件。
- 提供“重置”操作，一次清除所有筛选条件。
- 无匹配结果时展示明确空状态，不修改原始账单数据。

~~用户点击一条账单后进入编辑页面。~~ [2026-07-14 调整：不新增独立编辑页面。]

[2026-07-14 调整] 用户点击最近记录或全部账单中的一条记录后，在当前页面从底部打开编辑面板，可以修改：

- ~~类型。~~ [2026-07-14 删除：收支类型创建后不可修改。]
- 金额。
- 原收支类型下的分类。
- 日期。
- 备注。

[2026-07-14 补充后调整] 编辑账单修改的是用户原先通过“快速记一笔”录入的那条记录，也是“全部账单”列表中展示的同一条 `records` 数据，不会复制或创建另一套账单。最近记录中的账单也可以进入同一个编辑能力。

交互规则：

- 编辑面板复用“快速记一笔”的字段组件和校验逻辑，但拥有独立的编辑表单状态，不读写页面上方尚未提交的新增表单。
- 面板从屏幕底部弹出，覆盖当前页面下部并保留背景页面，不改变原列表滚动位置和筛选条件。
- 面板标题为“编辑账单”，主按钮为“保存修改”，同时提供关闭或取消操作。
- 点击遮罩、关闭按钮或取消操作时，如果编辑内容已变化，需要确认是否放弃修改；没有变化时直接关闭。
- 在全部账单二级页面保存后，面板关闭并原地更新对应记录，不关闭二级页面。
- ~~在今日记录保存后，面板关闭并原地更新对应记录，同时刷新概览、统计和预算。~~ [2026-07-14 调整：在最近记录保存后执行相同刷新。]
- 不采用列表行内展开编辑；完整表单字段较多，行内编辑会造成列表宽度不足和布局跳动。
- 编辑状态需要保存当前记录的 `clientId`，提交时执行更新而不是新增。

数据规则：

- 编辑时沿用新增账单的金额、分类和日期校验规则。
- ~~类型改变后，需要重新选择对应类型的有效分类。~~ [2026-07-14 删除：编辑时不再提供收支类型切换。]
- 编辑面板只加载原收支类型下的有效分类，保存时必须保留原记录的 `type`。
- 保存编辑时保留原 `clientId` 和 `createdAt`，仅更新业务字段和 `updatedAt`。
- 登录用户编辑后自动同步云端；同步失败时保留本地修改并明确提示。
- 编辑成功后刷新今日概览、全部账单、统计和预算进度。
- 删除继续采用软删除，不在编辑时物理删除云端记录。

### 5.2 统计功能

统计数据不单独长期存储，均根据账单记录和预算配置实时计算。

#### 5.2.1 按日统计

用户可以查看某一天的收支统计。

- [2026-07-16 补充] 按日统计的结余仅等于所选日期的收入减支出；按月统计的结余等于所选月份全部收入减全部支出，两者统计范围不同，界面分别标注“当日结余”和“本月结余”。

展示内容：

- 当日收入
- 当日支出
- 当日结余
- 当日记录明细

#### 5.2.2 按月统计

用户可以查看某个月的收支统计。

展示内容：

- 本月收入
- 本月支出
- 本月结余
- 月度每日支出趋势
- 月度支出分类占比

图表要求：

- 图表保持简洁，不做复杂交互。
- 至少支持月度每日支出趋势图。
- 至少支持月度支出分类占比图。

#### 5.2.3 月度统计增强

[2026-07-14 新增]

第二版月度统计增加：

- 支持切换任意已有账单月份。
- 展示本月收入、支出、结余和记账笔数。
- 展示与上月相比的收入、支出变化金额和变化比例。
- 展示本月结余与上月结余的差额，不展示结余变化百分比。
- 展示月度每日支出趋势。
- 展示支出分类占比和分类金额排名。
- 点击分类统计项后，可进入已按月份、支出类型和分类筛选的全部账单列表。

计算规则：

- 环比基准为所选月份的上一个自然月。
- 所选月份结余 = 所选月份收入 - 所选月份支出；上月结余 = 上月收入 - 上月支出。
- 结余环比差额 = 所选月份结余 - 上月结余。由于结余可能为 0 或负数，结余不计算环比百分比。
- 上月金额为 0 时不计算百分比，展示“暂无可比数据”。
- 分类占比继续按 `categoryId` 聚合，分类已停用时仍使用快照名称展示历史数据。
- 所有统计均由账单记录实时计算，不新增长期统计结果集合。

### 5.3 预算功能

#### 5.3.1 预算来源

预算由用户在小程序中手动设置。

第一版仅支持每月总预算，不支持分类预算。

#### 5.3.2 预算设置

用户可以在“我的”Tab 设置当前月份的总支出预算。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 月份 | 是 | 当前自然月 |
| 预算金额 | 是 | 必须大于 0 |

规则：

- 预算只和支出记录对比，不统计收入。
- 如果用户未设置本月预算，记账 Tab 不展示预算提醒，只展示设置入口。
- 第一版不做跨月预算结转。
- 第一版不要求下个月自动沿用上个月预算。

#### 5.3.3 预算提醒

预算提醒仅在小程序内展示。

提醒规则：

- 本月支出低于预算 80%：展示正常预算进度。
- 本月支出达到或超过预算 80%，但未超过预算：展示接近预算提醒。
- 本月支出超过预算 100%：展示超支提醒。

示例：

```text

本月已支出：1850 元
预算剩余：1150 元
使用进度：61.7%
```

## 6. 登录与数据同步

### 6.1 登录方式

小程序使用微信一键登录获取用户身份。

说明：

- [2026-07-17 新增] 当前小程序的应用标识固定为 `appKey=tinybookpro`。共享云环境中的所有业务数据必须携带该字段，查询和更新不得只依赖 `userId`。
- 微信登录仅用于识别用户身份。
- 小程序不自动读取微信支付账单。
- 用户在小程序中主动输入的数据才会被处理和统计。

### 6.2 未登录状态

未登录时：

- 用户可以正常新增、查看、删除记账记录。
- 数据存储在小程序本地缓存。
- 本地缓存是未登录状态下的数据源。
- 页面需要提示数据仅保存在本机。

### 6.3 登录后状态

登录后：

- 云端数据库是唯一可信数据源。
- 本地缓存仅用于页面加速、弱离线能力和减少重复拉取。
- `users` 集合保存当前用户的基础档案和最近登录时间。
- 账单原始数据、分类配置和预算配置需要保存到云端。
- 统计结果不作为长期可信数据存储。

### 6.4 本地数据同步到云端

用户登录后，如果本地存在未同步记录，自动同步到当前账号。

`clientId` 规则：

- 每条账单记录在客户端创建时生成一个稳定的 `clientId`。
- 未登录状态下创建的本地记录也必须包含 `clientId`。
- 登录后上传本地记录时，云端需要保存原始 `clientId`。
- 云端应使用 `userId + clientId` 作为去重依据，避免同一笔本地记录重复上传后生成多条云端记录。
- `clientId` 创建后不应因为编辑、同步或缓存刷新而改变。

同步规则：

- 登录成功后，将本地账单记录自动上传到云端。
- 上传前需要检查记录是否已有 `clientId`；如果历史本地记录缺少 `clientId`，需要先补齐再上传。
- 同步完成后，后续以云端数据为准。
- 本地缓存更新为云端数据的缓存副本。

### 6.5 登录后数据读取策略

用户进入小程序时：

1. 先读取本地缓存，快速展示页面。
2. 同时向云端拉取最新数据。
3. 云端数据返回后，更新本地缓存。
4. 根据最新账单和预算配置重新计算今日概览、月统计和预算进度。

~~第一版可以优先拉取当前月数据，避免一开始拉取所有历史数据。~~ [2026-07-14 调整：首屏仍只拉取当前月；全部账单、统计和导出根据时间范围分页按需读取。]

### 6.6 新增记录后的同步策略

~~登录后新增一笔记录时，第一版建议采用较简单的同步流程：~~ [2026-07-16 调整：第二版采用本地优先的弱网策略。]

1. 用户提交记账表单。
2. 校验通过后立即写入本地缓存并标记为待同步。
3. 网络可用时自动写入云端，成功后标记为已同步；失败时保留本地记录，不阻断记账。
4. 网络恢复后自动补同步待同步记录。
5. 根据本地最新数据重新计算今日概览、月统计和预算进度。

### 6.7 第二版查询与编辑同步策略

### 7.2 用户基础信息表：users

用户字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 云端记录 ID |
| appKey | 应用标识，固定为 tinybookpro |
| _openid | 云开发自动写入的用户 OpenID，用于数据权限判断 |
| userId | 用户 OpenID 的业务字段副本，用于集合查询和关联 |
| nickName | 用户授权后获得的微信昵称；未获得时为“微信用户” |
| avatarUrl | 用户授权后获得的微信头像地址；未获得时为空 |
| status | 用户状态，第一版固定为 active |
| createdAt | 首次登录并创建用户档案的时间 |
| lastLoginAt | 最近一次登录时间 |
| updatedAt | 用户档案最近更新时间 |

规则：

- ~~每个 OpenID 在 `users` 集合中只保留一条用户档案。~~ [2026-07-17 调整：每个 `appKey + userId` 组合只保留一条用户档案。]
- 首次登录时创建，后续登录更新 `lastLoginAt` 和 `updatedAt`。
- 仅在微信返回有效昵称或头像时更新对应字段，默认昵称不得覆盖已有的有效资料。
- 头像选择后先上传到云存储的 `apps/{appKey}/avatars/{userId}/` 目录，再将永久文件 ID 更新到 `users` 集合；昵称确认后同步更新 `users` 集合和本地用户缓存。
- 昵称长度限制为 1-20 个字符，保存前去除首尾空格并合并连续空格。
- [2026-07-16 新增] 昵称输入或粘贴超过 20 个字符时不得静默截断，需要显示实时字数和错误状态，失焦或提交时明确提示并禁止更新 `users` 集合。
- 昵称不得包含控制字符、换行符或零宽隐藏字符。
- 使用微信 `bindnicknamereview` 审核结果；仅审核通过时保存，审核未通过或超时均恢复原昵称。
- 不存储微信号、登录临时 code、会话密钥等敏感认证信息。
- `users` 集合权限设置为仅创建者可读写，前端写入时由云开发自动生成 `_openid`。
- 用户档案同步失败不阻断登录和记账，但需要记录错误以便排查。

### 7.7 本地缓存

本地建议缓存：

- 当前月账单记录。
- 分类列表。
- 本月预算配置。
- 用户登录状态。
- 上次同步时间 `lastSyncAt`。
- [2026-07-14 新增] 全部账单最近一次查询结果和分页游标，仅作为短期缓存。

- 今日收入。
- 今日支出。

- 分类占比。

这些统计数据应根据账单记录和预算配置实时计算。

## 9. 验收标准

### 9.1 记账能力

- 未登录用户可以新增支出记录。
- 未登录用户可以新增收入记录。
- 记录新增后，今日收入、今日支出、今日结余能正确更新。
- 用户可以删除当天记录。
- 删除记录后，相关统计能正确更新。

### 9.2 统计能力

- 用户可以查看某一天的收入、支出和结余。
- 用户可以查看某个月的收入、支出和结余。
- 月度支出分类占比计算正确。
- 月度每日支出趋势计算正确。

### 9.3 预算能力

- 用户可以设置本月总预算。
- 未设置预算时，不展示预算进度，只展示设置入口。
- 设置预算后，首页展示预算使用进度。
- 本月支出达到预算 80% 时，展示接近预算提醒。
- 本月支出超过预算 100% 时，展示超支提醒。

### 9.4 登录与同步

- 未登录时，数据仅保存在本地。
- 用户可以通过微信一键登录。
- 首次云端登录后，`users` 集合新增一条当前用户档案。
- 再次登录时不重复创建用户档案，并更新 `lastLoginAt`。
- 登录后，本地未同步记录自动上传到云端。
- 同步后，云端数据作为可信数据源。
- 再次进入小程序时，可以从云端恢复当前月数据。

### 9.5 数据边界

- 小程序不读取微信支付账单。
- 小程序只处理用户主动输入的数据。
- 统计结果不长期落库，避免账单与统计数据不一致。
