## 1. 产品概述

极简记账本 Pro 是一个面向个人用户的日常记账微信小程序。用户可以在不登录的情况下直接记录收入和支出，数据仅保存在本机；用户使用微信一键登录后，记账数据同步保存到云端数据库。

第一版目标是完成一个简洁、稳定、易用的个人记账闭环：快速记账、查看今日概览、查看日/月统计、设置每月总预算，并在小程序内展示预算使用提醒。

[2026-07-14 新增] 第二版目标是在不增加复杂账户体系的前提下，补齐账单管理、分类管理、统计分析和数据导出能力，让用户能够更方便地查找、修正和使用自己的历史记账数据。

[2026-07-18 新增] 第三版目标是在保留极简记账体验的前提下，通过快捷记账模板和周期账单降低重复录入成本，通过分类预算和年度统计提升预算控制与长期数据分析能力。

## 5. 功能需求

### 5.1 记账 Tab

#### 5.1.1 今日概览

用户打开小程序后，默认进入记账 Tab，页面展示今天的收支概览。

展示内容：

- 今日收入
- 今日支出
- 今日结余
- 本月支出
- 本月预算进度，用户已设置预算时展示

规则：

- 今日收入统计当天所有收入记录。
- 今日支出统计当天所有支出记录。
- 今日结余 = 今日收入 - 今日支出。
- 本月支出统计当前自然月内所有支出记录。
- 预算进度根据本月支出和本月总预算实时计算。

#### 5.1.2 快速记一笔

用户可以快速新增一条记账记录。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 类型 | 是 | 支出或收入 |
| 金额 | 是 | 必须大于 0 |
| 分类 | 是 | 第一版使用固定分类；第二版可选择系统分类或用户自定义分类 |
| 日期 | 是 | 默认今天，可修改 |
| 备注 | 否 | 用户可填写简单说明 |

交互规则：

- 默认类型为支出。
- 默认日期为当天。
- ~~金额不能为空，且必须为大于 0 的数字。~~ [2026-07-16 调整：金额必须大于 0、不超过 99,999,999.99，且最多保留两位小数。]
- [2026-07-16 新增] 备注最多 40 个字符；中文、英文、数字和表情均按用户可见字符计数。输入或粘贴超限内容时不得静默截断，需要显示实时字数和错误状态，保存时明确提示并阻止提交。
- ~~新增成功后，返回记账 Tab 并刷新今日概览、今日记录和预算进度。~~ [2026-07-14 调整：新增成功后刷新今日概览、最近记录和预算进度。]

记账 Tab 展示当天的记账记录列表。

~~[2026-07-14 补充] “今日记录”标题右侧增加蓝色文字入口“全部”，点击进入全部账单二级页面。~~ [2026-07-14 调整：“今日记录”改为“最近记录”，入口位置和文字“全部”保持不变。]

#### 5.1.5 全部账单

[2026-07-14 新增]

用户可以从记账 Tab 进入“全部账单”，查看当前账号或本机保存的历史记录。

[2026-07-14 补充后调整] “全部账单”不是新的数据类型，也不单独创建集合。它展示的是用户通过“快速记一笔”录入并保存在 `records` 集合或本地缓存中的全部有效收入、支出记录。最近记录、按日统计、按月统计和全部账单都是同一批账单数据的不同查询和展示方式。

单条账单记录的数据包括：

- 收支类型。
- 金额。
- 分类 ID 和分类名称快照。
- 记账日期。
- 备注。
- 创建时间和更新时间。
- 同步和软删除所需的内部字段；内部字段不直接展示给用户。

展示规则：

- 默认按记账日期倒序排列，同一天内按创建时间倒序排列。
- 列表按日期分组，日期标题展示当日收入、支出或净收支摘要。
- 单条记录展示类型、分类、金额、备注和记账时间。
- 已软删除的记录不展示。
- 首次加载最近 30 天或最近 100 条，继续滚动时分页加载，避免一次读取全部历史数据。

筛选条件：

- 时间范围：本月、上月、近 3 个月、自定义日期范围；单次最多覆盖连续 3 个自然月。
- 类型：全部、收入、支出。
- 分类：根据当前启用分类和历史账单分类生成选项。
- 备注关键词：按用户输入的关键词匹配备注。

筛选规则：

- 多个筛选条件同时存在时按交集查询。
- ~~开始和结束日期跨越的自然月份不得超过 3 个；日期选择器动态限制可选范围，查询前再次校验。~~ [2026-07-16 调整：日期选择器不互相限制可选日期，用户可以先后自由选择开始和结束日期；若开始日期晚于结束日期或跨越超过连续 3 个自然月，保留已选日期并在筛选区持续显示校验提示，不执行查询，调整为有效范围后自动查询。]
- 分类筛选使用两列级联选择：第一列为全部、支出、收入，第二列只展示所选类型下的分类；选择全部时第二列只显示全部分类。
- 顶部类型筛选按钮与分类级联选择保持同步，任一入口改变类型时清空不再适用的分类条件。
- 提供“重置”操作，一次清除所有筛选条件。
- 无匹配结果时展示明确空状态，不修改原始账单数据。

### 5.2 统计功能

统计数据不单独长期存储，均根据账单记录和预算配置实时计算。

#### 5.2.1 按日统计

用户可以查看某一天的收支统计。

- [2026-07-16 补充] 按日统计的结余仅等于所选日期的收入减支出；按月统计的结余等于所选月份全部收入减全部支出，两者统计范围不同，界面分别标注“当日结余”和“本月结余”。

展示内容：

- 当日收入
- 当日支出
- 当日结余
- 当日记录明细

#### 5.2.2 按月统计

用户可以查看某个月的收支统计。

展示内容：

- 本月收入
- 本月支出
- 本月结余
- 月度每日支出趋势
- 月度支出分类占比

图表要求：

- 图表保持简洁，不做复杂交互。
- 至少支持月度每日支出趋势图。
- 至少支持月度支出分类占比图。

#### 5.2.3 月度统计增强

[2026-07-14 新增]

第二版月度统计增加：

- 支持切换任意已有账单月份。
- 展示本月收入、支出、结余和记账笔数。
- 展示与上月相比的收入、支出变化金额和变化比例。
- 展示本月结余与上月结余的差额，不展示结余变化百分比。
- 展示月度每日支出趋势。
- 展示支出分类占比和分类金额排名。
- 点击分类统计项后，可进入已按月份、支出类型和分类筛选的全部账单列表。

计算规则：

- 环比基准为所选月份的上一个自然月。
- 所选月份结余 = 所选月份收入 - 所选月份支出；上月结余 = 上月收入 - 上月支出。
- 结余环比差额 = 所选月份结余 - 上月结余。由于结余可能为 0 或负数，结余不计算环比百分比。
- 上月金额为 0 时不计算百分比，展示“暂无可比数据”。
- 分类占比继续按 `categoryId` 聚合，分类已停用时仍使用快照名称展示历史数据。
- 所有统计均由账单记录实时计算，不新增长期统计结果集合。

### 5.3 预算功能

#### 5.3.1 预算来源

预算由用户在小程序中手动设置。

第一版仅支持每月总预算，不支持分类预算。

#### 5.3.2 预算设置

用户可以在“我的”Tab 设置当前月份的总支出预算。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 月份 | 是 | 当前自然月 |
| 预算金额 | 是 | 必须大于 0 |

规则：

- 预算只和支出记录对比，不统计收入。
- 如果用户未设置本月预算，记账 Tab 不展示预算提醒，只展示设置入口。
- 第一版不做跨月预算结转。
- 第一版不要求下个月自动沿用上个月预算。

#### 5.3.3 预算提醒

预算提醒仅在小程序内展示。

提醒规则：

- 本月支出低于预算 80%：展示正常预算进度。
- 本月支出达到或超过预算 80%，但未超过预算：展示接近预算提醒。
- 本月支出超过预算 100%：展示超支提醒。

示例：

```text
本月预算：3000 元
本月已支出：1850 元
预算剩余：1150 元
使用进度：61.7%
```

登录成功后自动创建或更新用户档案，并将本地未同步数据上传到当前账号，不提供手动同步入口。

- 导出过程发生失败时不得生成缺少部分记录的文件，应终止并提示用户重试。
- 导出前从云端分页读取所选时间范围内的全部有效账单。
- 不导出已软删除记录、OpenID、`userId`、`clientId` 等内部字段。
- CSV 字段需要正确处理逗号、双引号和换行，防止文件结构损坏。
- 文件生成成功后使用微信文件能力打开或分享，由用户主动保存。
- [2026-07-15 调整] CSV 文件使用 `wx.shareFileMessage` 分享，不使用仅支持特定文档类型的 `wx.openDocument`。
- ~~开发者工具无法完成 CSV 分享时应提示“CSV 已生成，请使用真机预览测试”，不得误报为数据导出失败。~~ [2026-07-17 删除：官方文档未说明真机调试不支持 `wx.shareFileMessage`，不能在没有原始错误的情况下归因于调试环境。]
- [2026-07-17 新增] 真机调试中分享失败时需要展示微信返回的原始 `errMsg`，并在控制台记录文件路径、文件大小、基础库版本、微信版本、平台和小程序运行版本；预览版或体验版用于发布前环境复验，不替代真机调试排错。
- [2026-07-17 新增] 导出采用两步交互：用户先点击“生成 CSV”完成数据查询和文件写入，生成成功后显示记录数和“分享 CSV”按钮；用户点击“分享 CSV”时必须立即调用 `wx.shareFileMessage`，调用前不得执行异步查询、异步写入或其他会使用户点击手势失效的任务。
- 无可导出记录时不生成空文件，展示明确提示。
- [2026-07-17 补充] 无数据、生成成功和生成失败的结果提示均在关闭加载状态后展示，避免提示被加载层立即覆盖。
- 第一阶段不支持 Excel 原生格式、自动定时导出或发送到邮箱。

#### 5.7.1 模板内容

每个模板包含：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 名称 | 是 | 便于识别模板，例如“早餐”“地铁”“工资” |

- 金额沿用账单金额规则：大于 0、不超过 99,999,999.99，最多两位小数；金额可以留空。
- 模板只保存相对稳定的信息，不保存固定记账日期；每次使用时日期默认当天。
- 模板分类被停用后，模板保留但不可直接使用，用户需要修改为有效分类后才能重新启用。
- 删除模板不影响已经通过该模板创建的账单。

#### 5.7.2 模板使用

- 记账 Tab 展示启用模板中的前 4-6 个，具体数量根据页面布局确定。
- 用户点击模板后，将类型、金额、分类和备注预填到“快速记一笔”表单，日期使用当天。
- 使用模板不得直接生成账单，用户仍需检查并点击“保存记录”。
- 模板预填不得覆盖用户已经在新增表单中输入但尚未保存的内容；表单已有内容时，切换模板前需要确认。
- 用户可以在预填后修改金额、分类、日期和备注，本次修改不自动回写模板。
- 模板管理支持新增、编辑、排序、停用、启用和删除。
- 未登录用户可以使用本机模板；登录后，本机模板按稳定 `clientId` 同步到当前账号并去重。

### 5.8 周期账单

[2026-07-18 新增，V3.0]

第三版支持用户创建周期账单计划，用于房租、工资、会员订阅等重复发生的收支。周期计划只生成待确认项，不自动写入正式账单。

#### 5.8.1 周期计划设置

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 名称 | 是 | 例如“每月房租”“每月工资” |
| 类型 | 是 | 支出或收入，创建后不可修改 |
| 金额 | 是 | 每期默认金额，确认时可修改 |
| 分类 | 是 | 系统分类或当前用户启用的自定义分类 |
| 备注 | 否 | 每期待记账项的默认备注 |
| 频率 | 是 | 每周、每月或每年 |
| 首次日期 | 是 | 第一期计划发生日期 |
| 结束方式 | 是 | 永不结束或指定结束日期 |
| 是否启用 | 是 | 暂停后不继续生成新的待确认项 |

规则：

- 每周按首次日期对应的星期重复。
- 每月按首次日期的日号重复；目标月份没有该日时，使用该月最后一天，例如 31 日在 2 月按月末处理。
- 每年按首次日期的月和日重复；2 月 29 日在非闰年按 2 月最后一天处理。
- 首次日期不得早于计划创建日期；结束日期存在时不得早于首次日期。
- 修改计划只影响尚未确认的未来期次，不修改已经生成的正式账单。
- 停用计划后保留历史状态，不生成新的待确认项；重新启用后仅补生成当前日期及之后的期次，不追溯停用期间的历史期次。
- 删除计划采用软删除，不删除由该计划生成的正式账单。

#### 5.8.2 待确认项生成与处理

- 小程序启动、进入记账 Tab 或进入周期账单页面时，检查已到期但尚未生成的周期账单。
- 对每个到期期次生成一条稳定的待确认项；同一计划、同一期次只能生成一次。
- 记账 Tab 只在存在待确认项时展示“待记账”入口和数量，不长期占用首屏空间。
- 用户可以对待确认项执行“确认记账”“修改后记账”或“跳过本期”。
- 确认后创建一条普通 `records` 账单，并记录来源计划和期次；后续编辑、删除规则与普通账单一致。
- 跳过只影响当前期次，不影响后续期次；跳过后可以在周期账单历史中查看，但第三版不提供恢复为待确认状态的入口。
- 多个待确认项需要逐条确认，第三版不提供批量确认，避免未经检查生成错误账单。
- 周期账单不依赖后台定时任务、订阅消息或服务通知；用户未打开小程序期间不发送提醒。

### 5.9 分类预算

[2026-07-18 新增，V3.0]

第三版在现有月度总预算基础上增加当前月份的支出分类预算。

#### 5.9.1 分类预算设置

- 用户从“我的”Tab 进入分类预算设置页。
- 预算月份固定为当前自然月。
- 用户可以为任一当前启用的支出分类设置一条预算，收入分类不能设置预算。
- 同一月份、同一分类只能存在一条有效预算；重复保存时更新原记录。
- 分类预算金额必须大于 0、不超过 99,999,999.99，最多两位小数。
- 用户可以修改或删除分类预算；删除预算不影响账单。
- 分类停用后，已设置的当月预算继续展示和统计，但不能再为后续月份新建预算。
- 第三版不自动复制上月分类预算，不支持预算结转，也不要求分类预算金额之和等于总预算。

#### 5.9.2 分类预算统计与提醒

- 分类已用金额等于当前月该 `categoryId` 下所有有效支出账单金额之和。
- 分类预算进度独立计算，不从总预算余额中扣减；总预算继续按全月支出计算。
- 提醒阈值沿用总预算规则：低于 80% 为正常，达到 80% 且未超过 100% 为接近预算，超过 100% 为超支。
- 记账 Tab 的总预算区域继续优先展示总预算，不在首屏同时平铺所有分类预算。
- 月度统计的分类支出列表中，对已设置预算的分类展示预算进度和剩余金额。
- 分类预算设置页展示每个已设置分类的预算、已用、剩余和状态。
- 分类预算提醒仅在小程序内展示，不发送微信订阅消息或服务通知。

### 5.10 年度统计

[2026-07-18 新增，V3.0]

第三版增加独立的年度统计二级页面，入口放在记账 Tab 的月度统计区域。

#### 5.10.1 年份范围与数据口径

- 默认展示当前自然年，用户可以切换存在账单数据的年份。
- 年度统计范围为所选年份 1 月 1 日至 12 月 31 日，不包含其他年份账单。
- 当前年份只统计截至当前日期已产生的有效账单，未来月份按无数据处理。
- 已软删除账单不参与统计；待确认或已跳过的周期账单不参与统计。
- 统计数据根据 `records` 实时聚合，不新增长期年度统计结果集合。

#### 5.10.2 展示内容

- 年度总收入、总支出、总结余和记账笔数。
- 12 个月收入与支出趋势；无账单月份显示为 0。
- 年度支出分类金额、占比和排名。
- 月均支出：年度支出除以参与平均的月份数。
- 日均支出：年度支出除以参与平均的自然日数。
- 支出最高月份及金额；没有支出数据时不展示该结论。
- 支出最高分类及金额；没有支出数据时不展示该结论。
- 点击月份可进入已按该自然月筛选的全部账单。
- 点击分类可进入已按该年份、支出类型和分类筛选的全部账单；若现有全部账单单次最多查询 3 个自然月，需要先调整查询范围能力或改为年度统计内的分类明细页，具体方案在开发前确认。

平均口径：

- 历史完整年份的月均支出按 12 个月计算，日均支出按该年实际自然日数计算。
- 当前年份的月均支出按从 1 月到当前月份的月份数计算，日均支出按从 1 月 1 日到当前日期的自然日数计算。
- 不按“有账单的月份或天数”计算平均值，避免漏记导致平均值失真。
- 第三版不提供年度同比、年度报告图片分享或消费人格分析。

## 6. 登录与数据同步

### 6.1 登录方式

小程序使用微信一键登录获取用户身份。

说明：

- [2026-07-17 新增] 当前小程序的应用标识固定为 `appKey=tinybookpro`。共享云环境中的所有业务数据必须携带该字段，查询和更新不得只依赖 `userId`。
- 微信登录仅用于识别用户身份。
- 小程序不自动读取微信支付账单。
- 用户在小程序中主动输入的数据才会被处理和统计。

### 6.2 未登录状态

未登录时：

- 用户可以正常新增、查看、删除记账记录。
- 数据存储在小程序本地缓存。
- 本地缓存是未登录状态下的数据源。
- 页面需要提示数据仅保存在本机。

### 6.3 登录后状态

登录后：

- 云端数据库是唯一可信数据源。
- 本地缓存仅用于页面加速、弱离线能力和减少重复拉取。
- `users` 集合保存当前用户的基础档案和最近登录时间。
- 账单原始数据、分类配置和预算配置需要保存到云端。
- 统计结果不作为长期可信数据存储。

### 6.4 本地数据同步到云端

用户登录后，如果本地存在未同步记录，自动同步到当前账号。

`clientId` 规则：

- 每条账单记录在客户端创建时生成一个稳定的 `clientId`。
- 未登录状态下创建的本地记录也必须包含 `clientId`。
- 登录后上传本地记录时，云端需要保存原始 `clientId`。
- 云端应使用 `userId + clientId` 作为去重依据，避免同一笔本地记录重复上传后生成多条云端记录。
- `clientId` 创建后不应因为编辑、同步或缓存刷新而改变。

同步规则：

- 登录成功后，将本地账单记录自动上传到云端。
- 上传前需要检查记录是否已有 `clientId`；如果历史本地记录缺少 `clientId`，需要先补齐再上传。
- 同步完成后，后续以云端数据为准。
- 本地缓存更新为云端数据的缓存副本。

### 6.5 登录后数据读取策略

用户进入小程序时：

1. 先读取本地缓存，快速展示页面。
2. 同时向云端拉取最新数据。
3. 云端数据返回后，更新本地缓存。
4. 根据最新账单和预算配置重新计算今日概览、月统计和预算进度。

~~第一版可以优先拉取当前月数据，避免一开始拉取所有历史数据。~~ [2026-07-14 调整：首屏仍只拉取当前月；全部账单、统计和导出根据时间范围分页按需读取。]

### 6.6 新增记录后的同步策略

~~登录后新增一笔记录时，第一版建议采用较简单的同步流程：~~ [2026-07-16 调整：第二版采用本地优先的弱网策略。]

1. 用户提交记账表单。
2. 校验通过后立即写入本地缓存并标记为待同步。
3. 网络可用时自动写入云端，成功后标记为已同步；失败时保留本地记录，不阻断记账。
4. 网络恢复后自动补同步待同步记录。
5. 根据本地最新数据重新计算今日概览、月统计和预算进度。

~~后续版本可以优化为先更新本地缓存，再后台同步云端，以提升弱网体验。~~ [2026-07-16 删除：该策略已在第二版实现。]

### 6.7 第二版查询与编辑同步策略

[2026-07-14 新增]

- 全部账单使用时间范围和分页查询，不一次拉取全部历史数据。
- 编辑账单继续以 `userId + clientId` 定位业务记录，避免设备缓存中的云端 `_id` 缺失时无法更新。
- 云端更新成功后更新本地缓存；云端失败时保留本地修改，并在后续登录或页面刷新时重试同步。
- 自定义分类和账单记录分别同步，账单保存前必须确认所选分类当前有效。
- ~~导出不依赖本地缓存，始终按选择的时间范围读取云端有效记录。~~ [2026-07-16 调整：导出优先读取云端，并合并所选月份本地缓存中的有效或待同步账单；云端暂时不可用时可使用本地缓存生成 CSV，避免弱网导致无结果。]

### 6.8 第三版数据同步策略

[2026-07-18 新增，V3.0]

- 快捷模板、周期计划、周期待确认项和分类预算均使用稳定 `clientId`，登录后按 `appKey + userId + clientId` 去重同步。
- 未登录用户可以在本机使用快捷模板、周期账单和分类预算；登录后自动同步到当前账号，不要求登录后才能使用第三版核心能力。
- 周期待确认项必须按“周期计划 ID + 计划期次日期”唯一生成，避免多设备或重复进入页面时生成多条相同期次。
- 用户确认周期账单时，待确认项状态更新和正式账单创建需要具备幂等性；重复提交不得产生重复账单。
- 第三版配置数据继续采用本地优先、后台同步策略；同步失败时保留本地变更并标记状态，不阻断用户继续记账。
- 年度统计根据所选年份分页读取云端账单并合并本地有效或待同步记录；云端失败时可展示本地可用结果，但必须提示数据可能不完整。

### 7.2 用户基础信息表：users

用户字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 云端记录 ID |
| appKey | 应用标识，固定为 tinybookpro |
| _openid | 云开发自动写入的用户 OpenID，用于数据权限判断 |
| userId | 用户 OpenID 的业务字段副本，用于集合查询和关联 |
| nickName | 用户授权后获得的微信昵称；未获得时为“微信用户” |
| avatarUrl | 用户授权后获得的微信头像地址；未获得时为空 |
| status | 用户状态，第一版固定为 active |
| createdAt | 首次登录并创建用户档案的时间 |
| lastLoginAt | 最近一次登录时间 |
| updatedAt | 用户档案最近更新时间 |

规则：

- ~~每个 OpenID 在 `users` 集合中只保留一条用户档案。~~ [2026-07-17 调整：每个 `appKey + userId` 组合只保留一条用户档案。]
- 首次登录时创建，后续登录更新 `lastLoginAt` 和 `updatedAt`。
- 仅在微信返回有效昵称或头像时更新对应字段，默认昵称不得覆盖已有的有效资料。
- 头像选择后先上传到云存储的 `apps/{appKey}/avatars/{userId}/` 目录，再将永久文件 ID 更新到 `users` 集合；昵称确认后同步更新 `users` 集合和本地用户缓存。
- 昵称长度限制为 1-20 个字符，保存前去除首尾空格并合并连续空格。
- [2026-07-16 新增] 昵称输入或粘贴超过 20 个字符时不得静默截断，需要显示实时字数和错误状态，失焦或提交时明确提示并禁止更新 `users` 集合。
- 昵称不得包含控制字符、换行符或零宽隐藏字符。
- 使用微信 `bindnicknamereview` 审核结果；仅审核通过时保存，审核未通过或超时均恢复原昵称。
- 不存储微信号、登录临时 code、会话密钥等敏感认证信息。
- `users` 集合权限设置为仅创建者可读写，前端写入时由云开发自动生成 `_openid`。
- 用户档案同步失败不阻断登录和记账，但需要记录错误以便排查。

### 7.3 账单记录表：records

账单记录字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 云端记录 ID |
| appKey | 应用标识，固定为 tinybookpro |
| clientId | 客户端创建记录时生成的稳定 ID，用于登录后同步去重 |
| userId | 用户 ID，登录后存在 |
| type | income 或 expense |
| amount | 金额 |
| categoryId | 分类 ID，系统分类关联 `categories.id`，自定义分类关联 `user_categories.id` |
| categoryNameSnapshot | 分类名称快照，用于分类被重命名或停用后仍能展示历史记录 |
| date | 记账日期 |
| remark | 备注 |
| sourceType | 账单来源，普通记账为 manual，周期账单确认为 recurring |
| sourcePlanClientId | 来源周期计划 `clientId`，普通记账时为空 |
| sourceOccurrenceDate | 来源周期期次日期，普通记账时为空 |
| createdAt | 创建时间 |
| updatedAt | 更新时间 |
| deletedAt | 删除时间，软删除时使用 |

规则：

- 新增账单时必须选择一个有效的 `categoryId`。
- `type` 需要和所选分类的 `type` 保持一致，例如支出账单只能选择支出分类。

## 9. 验收标准

### 9.1 记账能力

- 未登录用户可以新增支出记录。
- 未登录用户可以新增收入记录。
- 记录新增后，今日收入、今日支出、今日结余能正确更新。
- 用户可以删除当天记录。
- 删除记录后，相关统计能正确更新。

### 9.2 统计能力

- 用户可以查看某一天的收入、支出和结余。
- 用户可以查看某个月的收入、支出和结余。
- 月度支出分类占比计算正确。
- 月度每日支出趋势计算正确。

### 9.3 预算能力

- 用户可以设置本月总预算。
- 未设置预算时，不展示预算进度，只展示设置入口。
- 设置预算后，首页展示预算使用进度。
- 本月支出达到预算 80% 时，展示接近预算提醒。
- 本月支出超过预算 100% 时，展示超支提醒。

### 9.4 登录与同步

- 未登录时，数据仅保存在本地。
- 用户可以通过微信一键登录。
- 首次云端登录后，`users` 集合新增一条当前用户档案。
- 再次登录时不重复创建用户档案，并更新 `lastLoginAt`。
- 登录后，本地未同步记录自动上传到云端。
- 同步后，云端数据作为可信数据源。
- 再次进入小程序时，可以从云端恢复当前月数据。

### 9.5 数据边界

- 小程序不读取微信支付账单。
- 小程序只处理用户主动输入的数据。
- 统计结果不长期落库，避免账单与统计数据不一致。
