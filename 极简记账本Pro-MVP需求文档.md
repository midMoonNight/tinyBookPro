# 极简记账本 Pro 微信小程序 MVP 需求文档

## 1. 产品概述

极简记账本 Pro 是一个面向个人用户的日常记账微信小程序。用户可以在不登录的情况下直接记录收入和支出，数据仅保存在本机；用户使用微信一键登录后，记账数据同步保存到云端数据库。

第一版目标是完成一个简洁、稳定、易用的个人记账闭环：快速记账、查看今日概览、查看日/月统计、设置每月总预算，并在小程序内展示预算使用提醒。

## 2. MVP 目标

### 2.1 核心目标

- 用户打开小程序后，可以快速了解今天的收入、支出和结余。
- 用户可以快速记录一笔收入或支出，并选择固定分类。
- 用户可以查看按日、按月统计的收支情况。
- 用户可以设置每月总预算，并看到预算使用进度和超支提醒。
- 用户未登录时也能正常使用记账功能。
- 用户登录后，可以将本地数据同步到云端，后续以云端数据为准。

### 2.2 非目标

第一版暂不支持以下能力：

- 分类预算。
- 自定义收支分类。
- 多账本。
- 多人共享账本。
- 自动读取微信支付账单。
- 微信服务通知或模板消息提醒。
- 多币种。
- 发票、图片、附件管理。
- 复杂资产、负债、账户管理。

## 3. 用户与使用场景

### 3.1 目标用户

需要记录个人日常收入和支出的普通用户。

### 3.2 核心场景

- 用户消费后，打开小程序快速记录一笔支出。
- 用户收到工资、奖金或兼职收入后，记录一笔收入。
- 用户每天查看当天收支情况。
- 用户每月查看整体支出趋势和分类占比。
- 用户设置每月总预算，并关注本月支出是否接近或超过预算。
- 用户未登录时先试用，后续登录并同步本机数据。

## 4. 信息架构

小程序包含两个 Tab 页：

- 记账
- 我的

## 5. 功能需求

### 5.1 记账 Tab

#### 5.1.1 今日概览

用户打开小程序后，默认进入记账 Tab，页面展示今天的收支概览。

展示内容：

- 今日收入
- 今日支出
- 今日结余
- 本月支出
- 本月预算进度，用户已设置预算时展示

规则：

- 今日收入统计当天所有收入记录。
- 今日支出统计当天所有支出记录。
- 今日结余 = 今日收入 - 今日支出。
- 本月支出统计当前自然月内所有支出记录。
- 预算进度根据本月支出和本月总预算实时计算。

#### 5.1.2 快速记一笔

用户可以快速新增一条记账记录。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 类型 | 是 | 支出或收入 |
| 金额 | 是 | 必须大于 0 |
| 分类 | 是 | 使用固定分类 |
| 日期 | 是 | 默认今天，可修改 |
| 备注 | 否 | 用户可填写简单说明 |

交互规则：

- 默认类型为支出。
- 默认日期为当天。
- 金额不能为空，且必须为大于 0 的数字。
- 新增成功后，返回记账 Tab 并刷新今日概览、今日记录和预算进度。

#### 5.1.3 固定分类

第一版分类固定，不支持用户自定义。

支出分类：

- 餐饮
- 交通
- 购物
- 居住
- 娱乐
- 医疗
- 学习
- 其他

收入分类：

- 工资
- 奖金
- 兼职
- 投资
- 其他

#### 5.1.4 今日记录列表

记账 Tab 展示当天的记账记录列表。

展示内容：

- 类型
- 分类
- 金额
- 备注，存在时展示
- 记录时间或日期

第一版支持：

- 查看当天记录。
- 删除记录。

第一版暂不要求支持：

- 编辑记录。
- 搜索记录。
- 批量操作。

### 5.2 统计功能

统计数据不单独长期存储，均根据账单记录和预算配置实时计算。

#### 5.2.1 按日统计

用户可以查看某一天的收支统计。

展示内容：

- 当日收入
- 当日支出
- 当日结余
- 当日记录明细

#### 5.2.2 按月统计

用户可以查看某个月的收支统计。

展示内容：

- 本月收入
- 本月支出
- 本月结余
- 月度每日支出趋势
- 月度支出分类占比

图表要求：

- 图表保持简洁，不做复杂交互。
- 至少支持月度每日支出趋势图。
- 至少支持月度支出分类占比图。

### 5.3 预算功能

#### 5.3.1 预算来源

预算由用户在小程序中手动设置。

第一版仅支持每月总预算，不支持分类预算。

#### 5.3.2 预算设置

用户可以在“我的”Tab 设置当前月份的总支出预算。

字段：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 月份 | 是 | 当前自然月 |
| 预算金额 | 是 | 必须大于 0 |

规则：

- 预算只和支出记录对比，不统计收入。
- 如果用户未设置本月预算，记账 Tab 不展示预算提醒，只展示设置入口。
- 第一版不做跨月预算结转。
- 第一版不要求下个月自动沿用上个月预算。

#### 5.3.3 预算提醒

预算提醒仅在小程序内展示。

提醒规则：

- 本月支出低于预算 80%：展示正常预算进度。
- 本月支出达到或超过预算 80%，但未超过预算：展示接近预算提醒。
- 本月支出超过预算 100%：展示超支提醒。

示例：

```text
本月预算：3000 元
本月已支出：1850 元
预算剩余：1150 元
使用进度：61.7%
```

### 5.4 我的 Tab

我的 Tab 用于展示用户头像、昵称、登录入口和预算设置。

展示内容：

- 用户头像和昵称；未获得微信资料时展示默认信息。
- 已登录用户点击头像时使用微信 `chooseAvatar` 能力选择头像，点击昵称时使用 `type="nickname"` 能力选择或填写昵称，并更新用户档案。
- 微信一键登录入口。
- 本月预算设置入口。
- 退出登录入口。

未登录时提示：

```text
当前数据仅保存在本机，登录后可同步保存。
```

登录成功后自动创建或更新用户档案，并将本地未同步数据上传到当前账号，不提供手动同步入口。

## 6. 登录与数据同步

### 6.1 登录方式

小程序使用微信一键登录获取用户身份。

说明：

- 微信登录仅用于识别用户身份。
- 小程序不自动读取微信支付账单。
- 用户在小程序中主动输入的数据才会被处理和统计。

### 6.2 未登录状态

未登录时：

- 用户可以正常新增、查看、删除记账记录。
- 数据存储在小程序本地缓存。
- 本地缓存是未登录状态下的数据源。
- 页面需要提示数据仅保存在本机。

### 6.3 登录后状态

登录后：

- 云端数据库是唯一可信数据源。
- 本地缓存仅用于页面加速、弱离线能力和减少重复拉取。
- `users` 集合保存当前用户的基础档案和最近登录时间。
- 账单原始数据、分类配置和预算配置需要保存到云端。
- 统计结果不作为长期可信数据存储。

### 6.4 本地数据同步到云端

用户登录后，如果本地存在未同步记录，自动同步到当前账号。

`clientId` 规则：

- 每条账单记录在客户端创建时生成一个稳定的 `clientId`。
- 未登录状态下创建的本地记录也必须包含 `clientId`。
- 登录后上传本地记录时，云端需要保存原始 `clientId`。
- 云端应使用 `userId + clientId` 作为去重依据，避免同一笔本地记录重复上传后生成多条云端记录。
- `clientId` 创建后不应因为编辑、同步或缓存刷新而改变。

同步规则：

- 登录成功后，将本地账单记录自动上传到云端。
- 上传前需要检查记录是否已有 `clientId`；如果历史本地记录缺少 `clientId`，需要先补齐再上传。
- 同步完成后，后续以云端数据为准。
- 本地缓存更新为云端数据的缓存副本。

### 6.5 登录后数据读取策略

用户进入小程序时：

1. 先读取本地缓存，快速展示页面。
2. 同时向云端拉取最新数据。
3. 云端数据返回后，更新本地缓存。
4. 根据最新账单和预算配置重新计算今日概览、月统计和预算进度。

第一版可以优先拉取当前月数据，避免一开始拉取所有历史数据。

### 6.6 新增记录后的同步策略

登录后新增一笔记录时，第一版建议采用较简单的同步流程：

1. 用户提交记账表单。
2. 写入云端数据库。
3. 写入成功后更新本地缓存。
4. 重新计算今日概览、月统计和预算进度。

后续版本可以优化为先更新本地缓存，再后台同步云端，以提升弱网体验。

## 7. 数据设计

### 7.1 表名设计

微信云开发数据库中可以按以下集合名设计：

| 表名/集合名 | 说明 |
| --- | --- |
| users | 用户基础信息表 |
| records | 账单记录表 |
| categories | 记账分类表 |
| budgets | 月预算配置表 |

说明：

- 第一版分类由系统内置，用户不能自定义。
- 登录成功后按 OpenID 在 `users` 集合创建或更新一条用户档案。
- 即使第一版分类固定，账单记录也通过 `categoryId` 关联 `categories` 集合。
- 后续支持用户自定义分类时，可以在 `categories` 中新增用户维度的分类记录，不需要大改账单表结构。

### 7.2 用户基础信息表：users

用户字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 云端记录 ID |
| _openid | 云开发自动写入的用户 OpenID，用于数据权限判断 |
| userId | 用户 OpenID 的业务字段副本，用于集合查询和关联 |
| nickName | 用户授权后获得的微信昵称；未获得时为“微信用户” |
| avatarUrl | 用户授权后获得的微信头像地址；未获得时为空 |
| status | 用户状态，第一版固定为 active |
| createdAt | 首次登录并创建用户档案的时间 |
| lastLoginAt | 最近一次登录时间 |
| updatedAt | 用户档案最近更新时间 |

规则：

- 每个 OpenID 在 `users` 集合中只保留一条用户档案。
- 首次登录时创建，后续登录更新 `lastLoginAt` 和 `updatedAt`。
- 仅在微信返回有效昵称或头像时更新对应字段，默认昵称不得覆盖已有的有效资料。
- 头像选择后先上传到云存储，再将永久文件 ID 更新到 `users` 集合；昵称确认后同步更新 `users` 集合和本地用户缓存。
- 昵称长度限制为 1-20 个字符，保存前去除首尾空格并合并连续空格。
- 昵称不得包含控制字符、换行符或零宽隐藏字符。
- 使用微信 `bindnicknamereview` 审核结果；仅审核通过时保存，审核未通过或超时均恢复原昵称。
- 不存储微信号、登录临时 code、会话密钥等敏感认证信息。
- `users` 集合权限设置为仅创建者可读写，前端写入时由云开发自动生成 `_openid`。
- 用户档案同步失败不阻断登录和记账，但需要记录错误以便排查。

### 7.3 账单记录表：records

账单记录字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 云端记录 ID |
| clientId | 客户端创建记录时生成的稳定 ID，用于登录后同步去重 |
| userId | 用户 ID，登录后存在 |
| type | income 或 expense |
| amount | 金额 |
| categoryId | 分类 ID，关联 `categories.id` |
| categoryNameSnapshot | 分类名称快照，用于分类被重命名或停用后仍能展示历史记录 |
| date | 记账日期 |
| remark | 备注 |
| createdAt | 创建时间 |
| updatedAt | 更新时间 |
| deletedAt | 删除时间，软删除时使用 |

规则：

- 新增账单时必须选择一个有效的 `categoryId`。
- `type` 需要和所选分类的 `type` 保持一致，例如支出账单只能选择支出分类。
- 第一版固定分类也需要初始化到 `categories` 集合中。
- 统计分类占比时，以 `categoryId` 聚合；展示名称优先使用分类表名称，分类不存在时使用 `categoryNameSnapshot`。

### 7.4 分类表：categories

分类字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 分类 ID |
| userId | 用户 ID；系统内置分类可为空 |
| scope | system 或 user，第一版仅使用 system |
| type | income 或 expense |
| name | 分类名称 |
| icon | 分类图标，第一版可选 |
| sortOrder | 排序值 |
| isEnabled | 是否启用 |
| createdAt | 创建时间 |
| updatedAt | 更新时间 |
| deletedAt | 删除时间，软删除时使用 |

第一版系统内置分类：

| type | name |
| --- | --- |
| expense | 餐饮 |
| expense | 交通 |
| expense | 购物 |
| expense | 居住 |
| expense | 娱乐 |
| expense | 医疗 |
| expense | 学习 |
| expense | 其他 |
| income | 工资 |
| income | 奖金 |
| income | 兼职 |
| income | 投资 |
| income | 其他 |

规则：

- 第一版不提供用户新增、编辑、删除分类的入口。
- 后续支持用户自定义分类时，用户创建的分类写入同一张表，`scope` 为 `user`，并记录 `userId`。
- 分类被历史账单引用后，不建议物理删除；应使用 `isEnabled` 或 `deletedAt` 控制是否继续出现在新增记账选项中。

### 7.5 预算配置表：budgets

预算配置字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 云端记录 ID |
| userId | 用户 ID |
| month | 预算月份，格式如 2026-07 |
| amount | 月总预算金额 |
| createdAt | 创建时间 |
| updatedAt | 更新时间 |

### 7.6 本地缓存

本地建议缓存：

- 当前月账单记录。
- 分类列表。
- 本月预算配置。
- 用户登录状态。
- 上次同步时间 `lastSyncAt`。

不建议长期缓存：

- 今日收入。
- 今日支出。
- 本月收入。
- 本月支出。
- 预算剩余。
- 分类占比。
- 每日趋势。

这些统计数据应根据账单记录和预算配置实时计算。

## 8. 界面风格

### 8.1 视觉风格

- 整体风格简洁、清爽。
- 主色调以白色为主。
- 辅助色用于区分收入、支出、提醒状态。
- 信息层级清晰，避免复杂装饰。

### 8.2 页面原则

- 记账入口要明显。
- 今日概览要在首屏可见。
- 未登录提示要清楚，但不打断用户记账。
- 预算提醒要克制，只在接近预算或超支时强调。

## 9. 验收标准

### 9.1 记账能力

- 未登录用户可以新增支出记录。
- 未登录用户可以新增收入记录。
- 记录新增后，今日收入、今日支出、今日结余能正确更新。
- 用户可以删除当天记录。
- 删除记录后，相关统计能正确更新。

### 9.2 统计能力

- 用户可以查看某一天的收入、支出和结余。
- 用户可以查看某个月的收入、支出和结余。
- 月度支出分类占比计算正确。
- 月度每日支出趋势计算正确。

### 9.3 预算能力

- 用户可以设置本月总预算。
- 未设置预算时，不展示预算进度，只展示设置入口。
- 设置预算后，首页展示预算使用进度。
- 本月支出达到预算 80% 时，展示接近预算提醒。
- 本月支出超过预算 100% 时，展示超支提醒。

### 9.4 登录与同步

- 未登录时，数据仅保存在本地。
- 用户可以通过微信一键登录。
- 首次云端登录后，`users` 集合新增一条当前用户档案。
- 再次登录时不重复创建用户档案，并更新 `lastLoginAt`。
- 登录后，本地未同步记录自动上传到云端。
- 同步后，云端数据作为可信数据源。
- 再次进入小程序时，可以从云端恢复当前月数据。

### 9.5 数据边界

- 小程序不读取微信支付账单。
- 小程序只处理用户主动输入的数据。
- 统计结果不长期落库，避免账单与统计数据不一致。

## 10. 第一版待确认问题

以下问题不阻塞 MVP 需求，但进入开发前建议确认：

- 删除账单采用软删除还是物理删除。
- 统计入口放在记账 Tab 内，还是作为记账 Tab 的二级页面。
- 登录后用户取消同步本地数据时，本地未同步数据是否提供二次找回入口。
- 云端数据是否只拉取当前月，还是同时拉取最近 3 个月。
- 是否需要支持编辑已保存的记账记录。
